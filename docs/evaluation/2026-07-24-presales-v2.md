## 问题处理

本次没有修改或训练模型权重，而是采用“确定性规则优先，大模型处理模糊意图”的混合 Agent 方案：

1. 对“明确商品ID + 介绍/详情/特点/价格”等问题直接路由到 `get_product`。
2. 对“明确商品ID + 图片/视频/素材”等问题直接路由到 `search_media_assets`，并从问题中提取素材类型。
3. 模糊推荐、组合查询等开放问题继续交给大模型规划工具。
4. 为上述路由补充自动化回归测试，防止后续修改重新引入同类问题。

第二轮验证商品详情修复时，发现图片素材案例仍存在参数错误和重复调用。这说明真实评测的价值不只是证明系统能运行，还能暴露只靠人工演示不容易发现的稳定性问题。补充素材确定性路由后，再次运行完整评测集。

## 指标解释

- **平均耗时**：所有案例耗时相加后取平均，容易受到极慢请求影响。
- **P50**：一半请求的耗时不超过该数值，可近似理解为普通用户的典型等待时间。
- **P95**：95%的请求耗时不超过该数值，用于观察少数慢请求形成的尾延迟。
- **失败类型**：当前区分缺少工具、参数错误、空答案和运行异常，便于统计问题来源，而不是只保存一段错误文字。

## 结论与边界

本次结果证明当前版本在这16个固定案例上能够稳定选择预期工具，但不能推导出任意自然语言请求都能达到100%准确率。真实模型输出、网络状态和外部服务均存在波动，下一阶段应继续增加多轮指代、边界输入、对抗性提问和压力测试。
