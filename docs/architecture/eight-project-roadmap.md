# 电商 AI 八项目体系：真实进度矩阵

这份矩阵把最初架构图中的8个项目映射到当前仓库。状态只依据已经实现和验证的代码，不按设想计算。

| 项目 | 当前仓库对应能力 | 状态 | 简历建议 |
|---|---|---|---|
| 1. 商品交易自动化 Agent / Agent Runtime | FastAPI、Agent Loop、ToolRegistry、会话记忆、运行追踪、Demo | 简历MVP已完成 | 可作为平台底座或售前Agent的一部分重点描述 |
| 2. 商品交易售前实时咨询 Agent | 商品搜索、详情、库存、订单、RAG、多轮会话、评测 | 简历MVP已完成 | 当前最强业务项目，建议作为主项目 |
| 3. AI 数据中台 | 数据目录、8项质量规则、Staging、发布门禁、版本、激活、回滚 | 简历MVP已完成 | 描述为轻量数据中台，不称企业级大数据平台 |
| 4. AI 直播切片 Agent | 时间戳转写、高光规划、边界校验、素材登记 | 基础版已完成 | 描述为切片规划/高光识别；FFmpeg物理裁剪尚未实现 |
| 5. 自媒体内容运营 Agent | 多平台文案、事实约束、草稿、编辑、合规、人工审核、素材沉淀 | 简历MVP已完成 | 可作为第二个业务Agent描述 |
| 6. MCP & Agent 共享集群 | 5 Tools、1 Resource、1 Prompt、Streamable HTTP、Python Client | 简历MVP已完成 | 描述为AI公共能力与协议适配层 |
| 7. 多模态素材中心 | image/video/text元数据、商品绑定、标签、来源、统一检索 | 基础版已完成 | 描述为素材元数据中心；对象存储和生成式图片/视频尚未实现 |
| 8. 商品销售数据模型微调 / AI评测 | 固定评测集、工具准确率、运行历史、基线对比 | 评测完成，微调未实现 | 简历写Agent评测系统，不写模型微调项目 |

## 当前可形成的体系

```text
轻量数据中台
   ↓ 提供可信商品数据
售前 Agent ← ToolRegistry / RAG / MCP
   ↑                    ↑
内容运营 Agent → 人工审核 → 素材中心
直播切片 Agent ─────────→ 素材中心
   ↓
Agent评测 + 运行可观测
   ↓
Docker + GitHub Actions
```

## 当前简历策略

不建议把同一仓库机械拆成8段、每段都写成大型独立项目。更可信的表达是：

1. 一个“电商生态 AI Agent 平台”总项目；
2. 重点展开售前Agent、数据中台、内容/素材闭环、MCP公共服务；
3. 直播切片规划和评测作为子模块亮点；
4. 明确个人作品集和模拟业务数据边界。

这样既能体现体系化，又不会让面试官误以为应届生独立完成了8套生产级系统。

## 投简历前仍需完成

### 已完成

- 已初始化 Git 仓库并推送到 GitHub；
- 已获得 GitHub Actions 绿色运行记录，并在 README 展示状态徽章；
- 已检查 `.env`、SQLite、日志、虚拟环境和个人隐私没有进入版本控制。

### 必须完成

- 准备2到3分钟项目介绍和5到8分钟深挖版本；
- 准备一套稳定演示路径和截图。

### 有时间再做

- 接入 MinIO/TOS 等对象存储；
- 安装 FFmpeg，执行真实视频裁剪；
- 给 MCP 增加认证与限流；
- 增加 PostgreSQL、Redis 或异步任务队列；
- 在有真实训练数据、GPU和评测收益时再尝试模型微调。

## 当前不能写入简历的表述

- “完成企业级微服务集群”；
- “完成模型SFT/DPO微调并显著提升效果”；
- “支持大规模并发或百万级数据”；
- “已经完成真实视频自动裁剪和生成”；
- “项目产生真实商业收入或用户增长”。

以上能力没有真实代码、数据和测试支撑时不能使用。
